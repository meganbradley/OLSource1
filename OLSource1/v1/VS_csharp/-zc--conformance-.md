# -Zc (Conformance)
You can use the **/Zc** compiler option to specify standard or Microsoft-specific compiler behavior. Here are the **/Zc** compiler options:  
  
-   [/Zc:forScope](../VS_csharp/-zc-forscope--force-conformance-in-for-loop-scope-.md)  
  
-   [/Zc:wchar_t](../VS_csharp/-zc-wchar_t--wchar_t-is-native-type-.md)  
  
-   [/Zc:auto](../VS_csharp/-zc-auto--deduce-variable-type-.md)  
  
-   [/Zc:trigraphs](../VS_csharp/-zc-trigraphs--trigraphs-substitution-.md)  
  
-   [/Zc:rvalueCast](../VS_csharp/-zc-rvaluecast--enforce-type-conversion-rules-.md)  
  
-   [/Zc:strictStrings](../VS_csharp/-zc-strictstrings--disable-string-literal-type-conversion-.md)  
  
-   [/Zc:inline](../VS_csharp/-zc-inline--remove-unreferenced-comdat-.md)  
  
-   [/Zc:implicitNoexcept](../VS_csharp/-zc-implicitnoexcept--implicit-exception-specifiers-.md)  
  
 For more information about conformance issues in Visual C++, see [Nonstandard Behavior](../VS_csharp/nonstandard-behavior.md).
