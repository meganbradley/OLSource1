# Property Get-Let-Set are no longer supported; use the new Property declaration syntax
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder> are no longer supported; use the new <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> declaration syntax.  
  
 The syntax for declaring properties has changed. Properties are now defined within blocks.  
  
 **Error ID:** BC30808  
  
### To correct this error  
  
1.  Define properties in blocks of code that begin with the <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> keyword. End properties using the <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> construct.  
  
2.  Define <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> property procedures within property blocks with the <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> keyword. End <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> property procedures with the <CodeContentPlaceHolder>7\</CodeContentPlaceHolder> construct.  
  
3.  Define property <CodeContentPlaceHolder>8\</CodeContentPlaceHolder> procedures within property blocks with the <CodeContentPlaceHolder>9\</CodeContentPlaceHolder> keyword. End <CodeContentPlaceHolder>10\</CodeContentPlaceHolder> property procedures with the <CodeContentPlaceHolder>11\</CodeContentPlaceHolder> construct.  
  
4.  Use <CodeContentPlaceHolder>12\</CodeContentPlaceHolder> property procedures for all property assignments. <CodeContentPlaceHolder>13\</CodeContentPlaceHolder> property procedures are no longer necessary, or supported.
