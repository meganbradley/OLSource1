## Remarks  
 This class creates and positions control bars within the container application's document window. It also handles notifications generated by an embedded [COleResizeBar](../vs140/coleresizebar-class.md) object when the user resizes the in-place editing window.  
  
 For more information on using <CodeContentPlaceHolder>5\</CodeContentPlaceHolder>, see the article [Activation](../vs140/activation--c---.md).

##  \<a name="coleipframewnd__oncreatecontrolbars">\</a>  COleIPFrameWnd::OnCreateControlBars  
 The framework calls the <CodeContentPlaceHolder>8\</CodeContentPlaceHolder> function when an item is activated for in-place editing.  
  
<CodeContentPlaceHolder>2\</CodeContentPlaceHolder>  
### Parameters  
 *pWndFrame*  
 Pointer to the container application's frame window.  
  
 *pWndDoc*  
 Pointer to the container's document-level window. Can be **NULL** if the container is an SDI application.  
  
### Return Value  
 Nonzero on success; otherwise, 0.  
  
### Remarks  
 The default implementation does nothing. Override this function to perform any special processing required when control bars are created.  
  
##  \<a name="coleipframewnd__repositionframe">\</a>  COleIPFrameWnd::RepositionFrame  
 The framework calls the <CodeContentPlaceHolder>9\</CodeContentPlaceHolder> member function to lay out control bars and reposition the in-place editing window so all of it is visible.  
  
<CodeContentPlaceHolder>3\</CodeContentPlaceHolder>  
### Parameters  
 <CodeContentPlaceHolder>10\</CodeContentPlaceHolder>  
 Pointer to a <CodeContentPlaceHolder>11\</CodeContentPlaceHolder> structure or a <CodeContentPlaceHolder>12\</CodeContentPlaceHolder> object containing the in-place frame window's current position coordinates, in pixels, relative to the client area.  
  
 <CodeContentPlaceHolder>13\</CodeContentPlaceHolder>  
 Pointer to a <CodeContentPlaceHolder>14\</CodeContentPlaceHolder> structure or a <CodeContentPlaceHolder>15\</CodeContentPlaceHolder> object containing the in-place frame window's current clipping-rectangle coordinates, in pixels, relative to the client area.  
  
### Remarks  
 Layout of control bars in the container window differs from that performed by a non-OLE frame window. The non-OLE frame window calculates the positions of control bars and other objects from a given frame-window size, as in a call to [CFrameWnd::RecalcLayout](../vs140/cframewnd-class.md#cframewnd__recalclayout). The client area is what remains after space for control bars and other objects is subtracted. A <CodeContentPlaceHolder>16\</CodeContentPlaceHolder> window, on the other hand, positions toolbars in accordance with a given client area. In other words, <CodeContentPlaceHolder>17\</CodeContentPlaceHolder> works "from the outside in," whereas <CodeContentPlaceHolder>18\</CodeContentPlaceHolder> works "from the inside out."
