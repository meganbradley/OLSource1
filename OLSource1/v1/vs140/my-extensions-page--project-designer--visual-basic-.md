# My Extensions Page, Project Designer (Visual Basic)
Use the **My Extensions** page of the **Project Designer** to manage <CodeContentPlaceHolder>0\</CodeContentPlaceHolder> namespace extensions in your project. <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> namespace extensions enable you to customize the <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> namespace to add your own custom members. For information about creating custom <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> namespace extensions, see [Extending the My Namespace in Visual Basic](../vs140/extending-the-my-namespace-in-visual-basic.md).  
  
 To access the **My Extensions** page, double-click **My Project** for your project node in **Solution Explorer**. When the **Project Designer** appears, click the **My Extensions** tab.  
  
## UIElement List  
 The following options enable you to add or remove <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> namespace extensions in your project. A <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> namespace extension must first be installed as a Visual Studio Item template to be available to be added. For information about publishing and installing <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> namespace extensions, see [How to: Package and Deploy Custom My Extensions](../vs140/packaging-and-deploying-custom-my-extensions--visual-basic-.md).  
  
 **My namespace extensions**  
 This list shows all the <CodeContentPlaceHolder>7\</CodeContentPlaceHolder> namespace extensions installed in the project.  
  
 **Add Extension**  
 Click this button to add an installed <CodeContentPlaceHolder>8\</CodeContentPlaceHolder> namespace extension to your project. A list of all possible <CodeContentPlaceHolder>9\</CodeContentPlaceHolder> namespace extensions will appear. Select the <CodeContentPlaceHolder>10\</CodeContentPlaceHolder> namespace extension that you want to add to your project and click **OK** to add it.  
  
 **Remove Extension**  
 Select one or more references in the **My namespace extensions** list, and then click this button to remove the <CodeContentPlaceHolder>11\</CodeContentPlaceHolder> namespace extension from the project.
