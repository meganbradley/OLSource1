# C26112
warning C26112: Caller cannot hold any lock before calling \<func>.  
  
 The annotation <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> imposes a precondition that the caller must not hold any lock while it calls the function. Warning C26112 is issued when a function fails to release all locks before it calls another function.

 The following example generates warning C26112 because the <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> precondition is violated by the call to <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> within the locked section.
