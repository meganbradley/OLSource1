# CWnd::DlgDirListComboBox
Fills the list box of a combo box with a file or directory listing.  
  
## Syntax  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
#### Parameters  
 <CodeContentPlaceHolder>1\</CodeContentPlaceHolder>  
 Points to a null-terminated string that contains the path or filename. <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> modifies this string, so this data should not be in the form of a string literal. See the following "Remarks" section.  
  
 <CodeContentPlaceHolder>3\</CodeContentPlaceHolder>  
 Specifies the identifier of a combo box in a dialog box. If <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> is 0, <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> assumes that no combo box exists and does not attempt to fill one.  
  
 <CodeContentPlaceHolder>6\</CodeContentPlaceHolder>  
 Specifies the identifier of the static-text control used to display the current drive and directory. If <CodeContentPlaceHolder>7\</CodeContentPlaceHolder> is 0, <CodeContentPlaceHolder>8\</CodeContentPlaceHolder> assumes that no such text control is present.  
  
 <CodeContentPlaceHolder>9\</CodeContentPlaceHolder>  
 Specifies DOS file attributes of the files to be displayed. It can be any combination of the following values:  
  
-   **DDL_READWRITE** Read-write data files with no additional attributes.  
  
-   **DDL_READONLY** Read-only files.  
  
-   **DDL_HIDDEN** Hidden files.  
  
-   **DDL_SYSTEM** System files.  
  
-   **DDL_DIRECTORY** Directories.  
  
-   **DDL_ARCHIVE** Archives.  
  
-   **DDL_POSTMSGS CB_DIR** flag. If the **CB_DIR** flag is set, Windows places the messages generated by <CodeContentPlaceHolder>10\</CodeContentPlaceHolder> in the application's queue; otherwise, they are sent directly to the dialog-box procedure.  
  
-   **DDL_DRIVES** Drives. If the **DDL_DRIVES** flag is set, the **DDL_EXCLUSIVE** flag is set automatically. Therefore, to create a directory listing that includes drives and files, you must call <CodeContentPlaceHolder>11\</CodeContentPlaceHolder> twice: once with the **DDL_DRIVES** flag set and once with the flags for the rest of the list.  
  
-   **DDL_EXCLUSIVE** Exclusive bit. If the exclusive bit is set, only files of the specified type are listed; otherwise normal files and files of the specified type are listed.  
  
## Return Value  
 Specifies the outcome of the function. It is nonzero if a listing was made, even an empty listing. A 0 return value implies that the input string did not contain a valid search path.  
  
## Remarks  
 <CodeContentPlaceHolder>12\</CodeContentPlaceHolder> sends [CB_RESETCONTENT](http://msdn.microsoft.com/library/windows/desktop/bb775878) and [CB_DIR](http://msdn.microsoft.com/library/windows/desktop/bb775832) messages to the combo box. It fills the list box of the combo box specified by <CodeContentPlaceHolder>13\</CodeContentPlaceHolder> with the names of all files that match the path given by <CodeContentPlaceHolder>14\</CodeContentPlaceHolder>.  
  
 The <CodeContentPlaceHolder>15\</CodeContentPlaceHolder> parameter has the following form:  
  
 <CodeContentPlaceHolder>16\</CodeContentPlaceHolder>  
  
 In this example, <CodeContentPlaceHolder>17\</CodeContentPlaceHolder> is a drive letter, <CodeContentPlaceHolder>18\</CodeContentPlaceHolder> is a valid directory name, and *filename* is a valid filename that must contain at least one wildcard. The wildcards are a question mark (**?**), which means match any character, and an asterisk (**\***), which means match any number of characters.  
  
 If you specify a zero-length string for <CodeContentPlaceHolder>19\</CodeContentPlaceHolder>, the current directory will be used and <CodeContentPlaceHolder>20\</CodeContentPlaceHolder> will not be modified. If you specify only a directory name but do not include any file specification, the string will be changed to "*".  
  
 If <CodeContentPlaceHolder>21\</CodeContentPlaceHolder> includes a drive and/or directory name, the current drive and directory are changed to the designated drive and directory before the list box is filled. The text control identified by <CodeContentPlaceHolder>22\</CodeContentPlaceHolder> is also updated with the new drive and/or directory name.  
  
 After the combo-box list box is filled, <CodeContentPlaceHolder>23\</CodeContentPlaceHolder> is updated by removing the drive and/or directory portion of the path.
