# Type character cannot be used in a &#39;Sub&#39; declaration because a &#39;Sub&#39; doesn&#39;t return a value
A <CodeContentPlaceHolder>0\</CodeContentPlaceHolder> procedure, like a <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> procedure, is a separate procedure that can take arguments and perform a series of statements. Unlike a <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> procedure, a <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> does not return a value, and therefore cannot contain a type declaration.  
  
 **Error ID:** BC30303  
  
### To correct this error  
  
-   Change the <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> procedure to a <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> procedure.
