# 3.2.5 omp_test_lock and omp_test_nest_lock Functions
These functions attempt to set a lock but do not block execution of the thread. The format is as follows:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 The argument must point to an initialized lock variable. These functions attempt to set a lock in the same manner as <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> and <CodeContentPlaceHolder>2\</CodeContentPlaceHolder>, except that they do not block execution of the thread.  
  
 For a simple lock, the <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> function returns a nonzero value if the lock is successfully set; otherwise, it returns zero.  
  
 For a nestable lock, the <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> function returns the new nesting count if the lock is successfully set; otherwise, it returns zero.
