# __m64
## Microsoft Specific  
 The <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> data type is for use with the MMX and 3DNow! intrinsics, and is defined in xmmintrin.h.  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
## Remarks  
 You should not access the <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> fields directly. You can, however, see these types in the debugger. A variable of type <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> maps to the MM[0-7] registers.  
  
 Variables of type **_m64** are automatically aligned on 8-byte boundaries.  
  
 The <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> data type is not supported on [!INCLUDE[vcprx64](../vs140/includes/vcprx64_md.md)] processors. Applications that use __m64 as part of MMX intrinsics must be rewritten to use equivalent SSE and SSE2 intrinsics.
