# XML Documentation (F#)
You can produce documentation from triple-slash (///) code comments in F#. XML comments can precede declarations in code files (.fs) or signature (.fsi) files.  
  
## Generating Documentation from Comments  
 The support in F# for generating documentation from comments is the same as that in other .NET Framework languages. As in other .NET Framework languages, the [-doc compiler option](../vs140/compiler-options--fsharp-.md) enables you to produce an XML file that contains information that you can convert into documentation by using a tool such as Sandcastle. The documentation generated by using tools that are designed for use with assemblies that are written in other .NET Framework languages generally produce a view of the APIs that is based on the compiled form of F# constructs. Unless tools specifically support F#, documentation generated by these tools does not match the F# view of an API.  
  
 For more information about how to generate documentation from XML, see [XML Documentation Comments](../vs140/xml-documentation-comments--csharp-programming-guide-.md).  
  
## Recommended Tags  
 There are two ways to write XML documentation comments. One is to just write the documentation directly in a triple-slash comment, without using XML tags. If you do this, the entire comment text is taken as the summary documentation for the code construct that immediately follows. Use this method when you want to write only a brief summary for each construct. The other method is to use XML tags to provide more structured documentation. The second method enables you to specify separate notes for a short summary, additional remarks, documentation for each parameter and type parameter and exceptions thrown, and a description of the return value. The following table describes XML tags that are recognized in F# XML code comments.  
  
|Tag syntax|Description|  
|----------------|-----------------|  
|<CodeContentPlaceHolder>0\</CodeContentPlaceHolder> *text* <CodeContentPlaceHolder>1\</CodeContentPlaceHolder>|Specifies that *text* is code. This tag can be used by documentation generators to display text in a font that is appropriate for code.|  
|<CodeContentPlaceHolder>2\</CodeContentPlaceHolder> *text* <CodeContentPlaceHolder>3\</CodeContentPlaceHolder>|Specifies that *text* is a brief description of the program element. The description is usually one or two sentences.|  
|<CodeContentPlaceHolder>4\</CodeContentPlaceHolder> *text* <CodeContentPlaceHolder>5\</CodeContentPlaceHolder>|Specifies that *text* contains supplementary information about the program element.|  
|<CodeContentPlaceHolder>6\</CodeContentPlaceHolder> *name* <CodeContentPlaceHolder>7\</CodeContentPlaceHolder>  *description* <CodeContentPlaceHolder>8\</CodeContentPlaceHolder>|Specifies the name and description for a function or method parameter.|  
|<CodeContentPlaceHolder>9\</CodeContentPlaceHolder> *name* <CodeContentPlaceHolder>10\</CodeContentPlaceHolder>  *description*  <CodeContentPlaceHolder>11\</CodeContentPlaceHolder>|Specifies the name and description for a type parameter.|  
|<CodeContentPlaceHolder>12\</CodeContentPlaceHolder> *text* <CodeContentPlaceHolder>13\</CodeContentPlaceHolder>|Specifies that *text* describes the return value of a function or method.|  
|<CodeContentPlaceHolder>14\</CodeContentPlaceHolder> *type* <CodeContentPlaceHolder>15\</CodeContentPlaceHolder> *description* <CodeContentPlaceHolder>16\</CodeContentPlaceHolder>|Specifies the type of exception that can be generated and the circumstances under which it is thrown.|  
|<CodeContentPlaceHolder>17\</CodeContentPlaceHolder> *reference* <CodeContentPlaceHolder>18\</CodeContentPlaceHolder> *text* <CodeContentPlaceHolder>19\</CodeContentPlaceHolder>|Specifies an inline link to another program element. The *reference* is the name as it appears in the XML documentation file. The *text* is the text shown in the link.|  
|<CodeContentPlaceHolder>20\</CodeContentPlaceHolder> *reference* <CodeContentPlaceHolder>21\</CodeContentPlaceHolder>|Specifies a See Also link to the documentation for another type. The *reference* is the name as it appears in the XML documentation file. See Also links usually appear at the bottom of a documentation page.|  
|<CodeContentPlaceHolder>22\</CodeContentPlaceHolder> *text* <CodeContentPlaceHolder>23\</CodeContentPlaceHolder>|Specifies a paragraph of text. This is used to separate text inside the <CodeContentPlaceHolder>24\</CodeContentPlaceHolder> tag.|
