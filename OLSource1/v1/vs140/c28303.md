# C28303
warning C28303: For C++ reference-parameter \<parameter_name>, an ambiguous <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> operator was found on \<annotation>.  
  
 This warning similar to warning C28302 and is reported when an extra level of <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> is used on a parameter.  
  
 SAL2 does not require the use of an extra level of <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> when dealing with reference parameters. This particular annotation is ambiguous as to which level of dereference is intended to be annotated. It may be necessary to use <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> to reference the specific object to be annotated.  
  
## Example  
 The following code generates this warning because the use of <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> is ambiguous:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 The above annotation could be interpreted either as:  
  
-   a reference to an array (of n) pointers to integers (SAL1 interpretation)  
  
-   a reference to a pointer to an array (of n) integers (SAL2 interpretation)  
  
 Either of the following can correct this warning:  
  
<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>
