# C6255
warning C6255: _alloca indicates failure by raising a stack overflow exception. Consider using _malloca instead  
  
 This warning indicates that a call to <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> has been detected outside of local exception handling. <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> should always be called from within the protected range of an exception handler because it can raise a stack overflow exception on failure. If possible, instead of using <CodeContentPlaceHolder>4\</CodeContentPlaceHolder>, consider using <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> which is a more secure version of <CodeContentPlaceHolder>6\</CodeContentPlaceHolder>.

<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>
