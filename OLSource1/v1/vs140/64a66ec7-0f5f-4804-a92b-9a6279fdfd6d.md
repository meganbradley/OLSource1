# Cannot inherit interface &#39;&lt;interfacename1&gt;&#39; because the interface &#39;&lt;interfacename2&gt;&#39; from which it inherits could be identical to interface &#39;&lt;interfacename3&gt;&#39; from which the interface &#39;&lt;interfacename4&gt;&#39; inherits for some type arguments
A generic interface inherits from two or more generic interfaces, and two of the inheritances could conflict for certain values of type arguments.  
  
 The following statements can generate this error.  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 If <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> is constructed or implemented supplying the same type to both <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> and <CodeContentPlaceHolder>3\</CodeContentPlaceHolder>, it must inherit two versions of <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> with identical type arguments. Doing so would produce an ambiguity about which version to access.  
  
 **Error ID:** BC32122  
  
### To correct this error  
  
-   Change one of the type arguments supplied to the derived interface so that there is no conflict.  
  
     -or-  
  
-   Remove from the <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> statement one of the interfaces causing the potential inheritance or implementation conflict.
