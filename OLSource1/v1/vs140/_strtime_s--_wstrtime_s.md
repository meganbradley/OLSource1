# _strtime_s, _wstrtime_s
Copy the current time to a buffer. These are versions of [_strtime, _wstrtime](../vs140/_strtime--_wstrtime.md) with security enhancements as described in [Security Enhancements in the CRT](../vs140/security-features-in-the-crt.md).  
  
## Syntax  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
#### Parameters  
 [out] <CodeContentPlaceHolder>2\</CodeContentPlaceHolder>  
 A buffer, at least 10 bytes long, where the time will be written.  
  
 [in] <CodeContentPlaceHolder>3\</CodeContentPlaceHolder>  
 The size of the buffer.  
  
## Return Value  
 Zero if successful.  
  
 If an error condition occurs, the invalid parameter handler is invoked, as described in [Parameter Validation](../vs140/parameter-validation.md). The return value is an error code if there is a failure. Error codes are defined in ERRNO.H; see the following table for the exact errors generated by this function. For more information on error codes, see [errno Constants](../vs140/errno-constants.md).

## Security Issues  
 Passing in an invalid non-NULL value for the buffer will result in an access violation if the <CodeContentPlaceHolder>14\</CodeContentPlaceHolder> parameter is greater than 9.  
  
 Passing a value for <CodeContentPlaceHolder>15\</CodeContentPlaceHolder> that is greater than the actual size of the buffer will result in buffer overrun.  
  
## Remarks  
 These functions provide more secure versions of <CodeContentPlaceHolder>16\</CodeContentPlaceHolder> and <CodeContentPlaceHolder>17\</CodeContentPlaceHolder>. The <CodeContentPlaceHolder>18\</CodeContentPlaceHolder> function copies the current local time into the buffer pointed to by <CodeContentPlaceHolder>19\</CodeContentPlaceHolder>*.* The time is formatted as <CodeContentPlaceHolder>20\</CodeContentPlaceHolder> where <CodeContentPlaceHolder>21\</CodeContentPlaceHolder> is two digits representing the hour in 24-hour notation, <CodeContentPlaceHolder>22\</CodeContentPlaceHolder> is two digits representing the minutes past the hour, and <CodeContentPlaceHolder>23\</CodeContentPlaceHolder> is two digits representing seconds. For example, the string <CodeContentPlaceHolder>24\</CodeContentPlaceHolder> represents 23 minutes and 44 seconds past 6 P.M. The buffer must be at least 9 bytes long; the actual size is specified by the second parameter.  
  
 <CodeContentPlaceHolder>25\</CodeContentPlaceHolder> is a wide-character version of <CodeContentPlaceHolder>26\</CodeContentPlaceHolder>; the argument and return value of <CodeContentPlaceHolder>27\</CodeContentPlaceHolder> are wide-character strings. These functions behave identically otherwise.  
  
 In C++, using these functions is simplified by template overloads; the overloads can infer buffer length automatically (eliminating the need to specify a size argument) and they can automatically replace older, non-secure functions with their newer, secure counterparts. For more information, see [Secure Template Overloads](../vs140/secure-template-overloads.md).
