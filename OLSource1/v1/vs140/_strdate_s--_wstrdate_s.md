# _strdate_s, _wstrdate_s
Copy the current system date to a buffer. These are versions of [_strdate, _wstrdate](../vs140/_strdate--_wstrdate.md) with security enhancements as described in [Security Enhancements in the CRT](../vs140/security-features-in-the-crt.md).  
  
## Syntax  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
#### Parameters  
 [out] <CodeContentPlaceHolder>1\</CodeContentPlaceHolder>  
 A pointer to a buffer which will be filled in with the formatted date string.  
  
 [in] <CodeContentPlaceHolder>2\</CodeContentPlaceHolder>  
 Size of the buffer.  
  
## Return Value  
 Zero if successful. The return value is an error code if there is a failure. Error codes are defined in ERRNO.H; see table below for the exact errors generated by this function. For more information on error codes, see [errno](../vs140/errno-constants.md).

## Security Issues  
 Passing in an invalid non <CodeContentPlaceHolder>15\</CodeContentPlaceHolder> value for the buffer will result in an access violation if the <CodeContentPlaceHolder>16\</CodeContentPlaceHolder> parameter is greater than 9.  
  
 Passing values for size that is greater than the actual size of the <CodeContentPlaceHolder>17\</CodeContentPlaceHolder> will result in buffer overrun.  
  
## Remarks  
 These functions provide more secure versions of <CodeContentPlaceHolder>18\</CodeContentPlaceHolder> and <CodeContentPlaceHolder>19\</CodeContentPlaceHolder>. The <CodeContentPlaceHolder>20\</CodeContentPlaceHolder> function copies the current system date to the buffer pointed to by <CodeContentPlaceHolder>21\</CodeContentPlaceHolder>, formatted <CodeContentPlaceHolder>22\</CodeContentPlaceHolder>/<CodeContentPlaceHolder>23\</CodeContentPlaceHolder>/<CodeContentPlaceHolder>24\</CodeContentPlaceHolder>, where <CodeContentPlaceHolder>25\</CodeContentPlaceHolder> is two digits representing the month, <CodeContentPlaceHolder>26\</CodeContentPlaceHolder> is two digits representing the day, and <CodeContentPlaceHolder>27\</CodeContentPlaceHolder> is the last two digits of the year. For example, the string <CodeContentPlaceHolder>28\</CodeContentPlaceHolder> represents December 5, 1999. The buffer must be at least 9 characters long.  
  
 <CodeContentPlaceHolder>29\</CodeContentPlaceHolder> is a wide-character version of <CodeContentPlaceHolder>30\</CodeContentPlaceHolder>; the argument and return value of <CodeContentPlaceHolder>31\</CodeContentPlaceHolder> are wide-character strings. These functions behave identically otherwise.  
  
 If <CodeContentPlaceHolder>32\</CodeContentPlaceHolder> is a <CodeContentPlaceHolder>33\</CodeContentPlaceHolder> pointer, or if <CodeContentPlaceHolder>34\</CodeContentPlaceHolder> is less than 9 characters, the invalid parameter handler is invoked, as described in [Parameter Validation](../vs140/parameter-validation.md). If execution is allowed to continue, these functions return -1 and set <CodeContentPlaceHolder>35\</CodeContentPlaceHolder> to <CodeContentPlaceHolder>36\</CodeContentPlaceHolder> if the buffer is <CodeContentPlaceHolder>37\</CodeContentPlaceHolder> or if <CodeContentPlaceHolder>38\</CodeContentPlaceHolder> is less than or equal to 0, or set <CodeContentPlaceHolder>39\</CodeContentPlaceHolder> to <CodeContentPlaceHolder>40\</CodeContentPlaceHolder> if <CodeContentPlaceHolder>41\</CodeContentPlaceHolder> is less than 9.  
  
 In C++, using these functions is simplified by template overloads; the overloads can infer buffer length automatically (eliminating the need to specify a size argument) and they can automatically replace older, non-secure functions with their newer, secure counterparts. For more information, see [Secure Template Overloads](../vs140/secure-template-overloads.md).
