# C6215
warning C6215: cast between semantically different integer types: a Boolean type to HRESULT  
  
 This warning indicates that a Boolean is being cast to an <CodeContentPlaceHolder>2\</CodeContentPlaceHolder>. Boolean types indicate success by a non-zero value, whereas success (<CodeContentPlaceHolder>3\</CodeContentPlaceHolder>) in <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> is indicated by a value of 0. Casting a Boolean type to an <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> and then using it in a test expression will yield an incorrect result.  
  
 This warning frequently occurs when a Boolean is used as an argument to <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> or <CodeContentPlaceHolder>7\</CodeContentPlaceHolder> macro, which explicitly casts their arguments to an <CodeContentPlaceHolder>8\</CodeContentPlaceHolder>.  
  
## Example  
 The following code generates this warning:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 Generally, the <CodeContentPlaceHolder>9\</CodeContentPlaceHolder> or <CodeContentPlaceHolder>10\</CodeContentPlaceHolder> macros should only be applied to <CodeContentPlaceHolder>11\</CodeContentPlaceHolder>.  
  
 To correct this warning, use the following code:
