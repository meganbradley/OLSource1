# ConcRT_EventType Enumeration
The types of events that can be traced using the tracing functionality offered by the Concurrency Runtime.  
  
## Syntax  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
## Members  
  
### Values  
  
|Name|Description|  
|----------|-----------------|  
|<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>|An event type that represents the act of a attaching to a scheduler.|  
|<CodeContentPlaceHolder>2\</CodeContentPlaceHolder>|An event type that represents the act of a context blocking.|  
|<CodeContentPlaceHolder>3\</CodeContentPlaceHolder>|An event type that represents the act of a detaching from a scheduler.|  
|<CodeContentPlaceHolder>4\</CodeContentPlaceHolder>|An event type that marks the beginning of a start/end event pair.|  
|<CodeContentPlaceHolder>5\</CodeContentPlaceHolder>|An event type used for miscellaneous events.|  
|<CodeContentPlaceHolder>6\</CodeContentPlaceHolder>|An event type that represents the act of a context becoming idle.|  
|<CodeContentPlaceHolder>7\</CodeContentPlaceHolder>|An event type that marks the beginning of a start/end event pair.|  
|<CodeContentPlaceHolder>8\</CodeContentPlaceHolder>|An event type that represents the act of unblocking a context.|  
|<CodeContentPlaceHolder>9\</CodeContentPlaceHolder>|An event type that represents the act of a context yielding.|
