# C6273
Warning 6273 - non-integer passed as parameter \<number> when integer is required in call to \<function>: if a pointer value is being passed, %p should be used  
  
 This warning indicates that the format string specifies an integer, for example, a <CodeContentPlaceHolder>3\</CodeContentPlaceHolder>, length or precedence specification for <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> but a non-integer such as a <CodeContentPlaceHolder>5\</CodeContentPlaceHolder>, string, or <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> is being passed as a parameter. This defect is likely to result in incorrect output.  
  
## Example  
 The following code generates this warning because an integer is required instead of a <CodeContentPlaceHolder>7\</CodeContentPlaceHolder> to <CodeContentPlaceHolder>8\</CodeContentPlaceHolder>function:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 The following code uses an integer cast to correct this warning:  
  
<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>  
 The following code uses safe string manipulation function, <CodeContentPlaceHolder>9\</CodeContentPlaceHolder>, to correct this warning:  
  
<CodeContentPlaceHolder>2\</CodeContentPlaceHolder>  
 This warning is not applicable on Windows 9x and Windows NT version 4 because %p is not supported on these platforms.
