# C26111
warning C26111: Caller failing to release lock \<lock> before calling function \<func>.  
  
 The annotation <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> imposes a precondition that the lock count for the specified lock cannot be greater than zero when the function is called. Warning C26111 is issued when a function fails to release the lock before it calls another function.

 The following example generates warning C26111 because the <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> precondition is violated by the call to <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> within the locked section.
