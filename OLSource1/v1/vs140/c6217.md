# C6217
warning C6217: Implicit cast between semantically different integer types: testing HRESULT with 'not'. Consider using [SUCCEEDED](http://go.microsoft.com/fwlink/?LinkId=92738) or [FAILED](ms-help://MS.VSCC.2003/MS.MSDNQTR.2003FEB.1033/com/htm/error_899v.htm) macro instead.  
  
 This warning indicates that an <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> is being tested with the not (<CodeContentPlaceHolder>3\</CodeContentPlaceHolder>) operator. A success (<CodeContentPlaceHolder>4\</CodeContentPlaceHolder>) in <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> is indicated by a value of 0. However, 0 indicates failure for a Boolean type. Testing <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> with the not operator (<CodeContentPlaceHolder>7\</CodeContentPlaceHolder>) to determine which code block to run can cause following the wrong code path. This will lead to unwanted results.  
  
## Example  
 The following code generates this warning because the not operator is used to determine success or failure of an <CodeContentPlaceHolder>8\</CodeContentPlaceHolder> value. In this case, wrong code path is executed because <CodeContentPlaceHolder>9\</CodeContentPlaceHolder> runs the failure code:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 To correct this warning, the following code uses <CodeContentPlaceHolder>10\</CodeContentPlaceHolder> macro to look for failure:  
  
<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>  
 For this warning, the <CodeContentPlaceHolder>11\</CodeContentPlaceHolder> type is equivalent to <CodeContentPlaceHolder>12\</CodeContentPlaceHolder>.  
  
 The typical success value of HRESULT (<CodeContentPlaceHolder>13\</CodeContentPlaceHolder>) is <CodeContentPlaceHolder>14\</CodeContentPlaceHolder> when it is tested as a Boolean.  
  
 To verify whether <CodeContentPlaceHolder>15\</CodeContentPlaceHolder> is a success, use the <CodeContentPlaceHolder>16\</CodeContentPlaceHolder> macro instead.
