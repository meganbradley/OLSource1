# Compiler Warning (level 1) CS1692
Invalid number  
  
 A number of preprocessor directives, such as <CodeContentPlaceHolder>1\</CodeContentPlaceHolder> and <CodeContentPlaceHolder>2\</CodeContentPlaceHolder>, use numbers as parameters. One of these numbers is invalid because it is too big, in the wrong format, contains illegal characters, and so on. To correct this error, correct the number.  
  
## Example  
 The following example generates CS1692.  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>
