# &#39;Microsoft.VisualBasic.ComClassAttribute&#39; cannot be applied to &#39;&lt;classname&gt;&#39; because it is not declared &#39;Public&#39;
A class is declared with \<xref:Microsoft.VisualBasic.ComClassAttribute*>, but its declaration does not specify <CodeContentPlaceHolder>0\</CodeContentPlaceHolder>.  
  
 To be eligible for COM interop, a .NET Framework class must satisfy the following requirements:  
  
-   It must be <CodeContentPlaceHolder>1\</CodeContentPlaceHolder>, all its containers must be <CodeContentPlaceHolder>2\</CodeContentPlaceHolder>, and it must expose at least one <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> member.  
  
-   It must not be *abstract*, that is, it must not be declared with <CodeContentPlaceHolder>4\</CodeContentPlaceHolder>.  
  
-   It must not be generic or be declared within a generic container type.  
  
 **Error ID:** BC32509  
  
### To correct this error  
  
-   Add the <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> keyword to the class declaration.  
  
     -or-  
  
-   If the class or its containing element cannot be <CodeContentPlaceHolder>6\</CodeContentPlaceHolder>, then remove \<xref:Microsoft.VisualBasic.ComClassAttribute*> from the class declaration. You cannot expose it to COM.
