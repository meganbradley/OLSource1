# MarkupCompilePass1 Task
The \<xref:Microsoft.Build.Tasks.Windows.MarkupCompilePass1*> task converts non-localizable [!INCLUDE[TLA#tla_xaml](../vs140/includes/tlasharptla_xaml_md.md)] project files to compiled binary format.

## Remarks  
 The \<xref:Microsoft.Build.Tasks.Windows.MarkupCompilePass1*> task typically compiles [!INCLUDE[TLA2#tla_xaml](../vs140/includes/tla2sharptla_xaml_md.md)] into binary format and generates code files. If a [!INCLUDE[TLA2#tla_xaml](../vs140/includes/tla2sharptla_xaml_md.md)] file contains references to types that are defined in the same project, its compilation to binary format is deferred by **MarkupCompilePass1** to a second markup compilation pass (**MarkupCompilePass2**). Such files must have their compilation deferred because they must wait until the referenced locally-defined types are compiled. However, if a [!INCLUDE[TLA2#tla_xaml](../vs140/includes/tla2sharptla_xaml_md.md)] file has an <CodeContentPlaceHolder>33\</CodeContentPlaceHolder> attribute, \<xref:Microsoft.Build.Tasks.Windows.MarkupCompilePass1*> generates the language-specific code file for it.  
  
 A [!INCLUDE[TLA2#tla_xaml](../vs140/includes/tla2sharptla_xaml_md.md)] file is localizable if it contains elements that use the <CodeContentPlaceHolder>34\</CodeContentPlaceHolder> attribute:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 A [!INCLUDE[TLA2#tla_xaml](../vs140/includes/tla2sharptla_xaml_md.md)] file references a locally-defined type when it declares an [!INCLUDE[TLA#tla_xml](../vs140/includes/tlasharptla_xml_md.md)] namespace that uses the <CodeContentPlaceHolder>35\</CodeContentPlaceHolder> value to refer to a namespace in the current project:  
  
<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>  
 If any [!INCLUDE[TLA2#tla_xaml](../vs140/includes/tla2sharptla_xaml_md.md)] file is localizable, or references a locally-defined type, a second markup compilation pass is required, which requires running the [GenerateTemporaryTargetAssembly Task](../vs140/generatetemporarytargetassembly-task.md) and then the [MarkupCompilePass2 Task](../vs140/markupcompilepass2-task.md).

Instead, the [GenerateTemporaryTargetAssembly Task](../vs140/generatetemporarytargetassembly-task.md) is executed and is followed by the [MarkupCompilePass2 Task](../vs140/markupcompilepass2-task.md).
