# C6216
warning C6216: compiler-inserted cast between semantically different integral types: a Boolean type to HRESULT  
  
 This warning indicates that a Boolean is being used as an <CodeContentPlaceHolder>2\</CodeContentPlaceHolder> without being explicitly cast. Boolean types indicate success by a non-zero value; success (<CodeContentPlaceHolder>3\</CodeContentPlaceHolder>) in <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> is indicated by a value of 0. The typical failure value for functions that return a Boolean false is a success status when it is tested as an <CodeContentPlaceHolder>5\</CodeContentPlaceHolder>. This is likely to lead to incorrect results.  
  
## Example  
 The following code generates this warning:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
 To correct this warning, use the following code:  
  
<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>  
 For this warning, the <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> type is equivalent to <CodeContentPlaceHolder>7\</CodeContentPlaceHolder>.  
  
 For more information, see [SUCCEEDED Macro](http://go.microsoft.com/fwlink/?LinkId=92738) and [FAILED Macro](http://go.microsoft.com/fwlink/?LinkId=180875).
