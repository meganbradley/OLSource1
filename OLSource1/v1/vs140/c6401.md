# C6401
warning C6401: Using \<function name> in a default locale to perform a case-insensitive compare to constant string \< string name>. Yields unexpected results in non-English locales  
  
 This warning indicates that a case-insensitive comparison to a constant string is being performed when specifying the default locale; usually, a locale-independent comparison was intended.  
  
 The typical consequence of this defect is incorrect behavior in non-English speaking locales. For example, in Turkish, ".gif" will not match ".GIF"; in Vietnamese, "LogIn" will not match "LOGIN".  
  
 The <CodeContentPlaceHolder>3\</CodeContentPlaceHolder> function takes a locale as an argument; however, passing in a default locale, for example, the constant <CodeContentPlaceHolder>4\</CodeContentPlaceHolder>, will cause different behaviors in different locales, depending on the user's default. Usually, case-insensitive comparisons against a constant string should be performed in a locale-independent comparison.  
  
 To perform a locale-independent comparison using <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> on Windows XP, the first parameter should be the constant <CodeContentPlaceHolder>6\</CodeContentPlaceHolder>; for example, to perform a locale-independent test for whether <CodeContentPlaceHolder>7\</CodeContentPlaceHolder> matches file1.gif ignoring upper/lower case differences, use a call such as:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
## Example  
 The following code generates this warning:  
  
<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>  
 To correct this warning, use the following code:  
  
<CodeContentPlaceHolder>2\</CodeContentPlaceHolder>
