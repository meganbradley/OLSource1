# Compiler Error C2289
same type qualifier used more than once  
  
 A type declaration or definition uses a type qualifier (<CodeContentPlaceHolder>1\</CodeContentPlaceHolder>, <CodeContentPlaceHolder>2\</CodeContentPlaceHolder>, <CodeContentPlaceHolder>3\</CodeContentPlaceHolder>, or <CodeContentPlaceHolder>4\</CodeContentPlaceHolder>) more than once, causing an error under ANSI compatibility (**/Za**).  
  
 The following sample generates C2286:  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>
