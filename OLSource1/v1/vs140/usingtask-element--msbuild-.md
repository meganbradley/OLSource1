# UsingTask Element (MSBuild)
Maps the task that is referenced in a [Task](../vs140/task-element--msbuild-.md) element to the assembly that contains the task implementation.  
  
 \<Project>  
 \<UsingTask>  
  
## Syntax  
  
<CodeContentPlaceHolder>0\</CodeContentPlaceHolder>  
## Attributes and Elements  
 The following sections describe attributes, child elements, and parent elements.  
  
### Attributes  
  
|Attribute|Description|  
|---------------|-----------------|  
|<CodeContentPlaceHolder>3\</CodeContentPlaceHolder>|Either the <CodeContentPlaceHolder>4\</CodeContentPlaceHolder> attribute or the <CodeContentPlaceHolder>5\</CodeContentPlaceHolder> attribute is required.\<br />\<br /> The name of the assembly to load. The <CodeContentPlaceHolder>6\</CodeContentPlaceHolder> attribute accepts strong-named assemblies, although strong-naming is not required. Using this attribute is equivalent to loading an assembly by using the \<xref:System.Reflection.Assembly.Load*> method in .NET.\<br />\<br /> You cannot use this attribute if the <CodeContentPlaceHolder>7\</CodeContentPlaceHolder> attribute is used.|  
|<CodeContentPlaceHolder>8\</CodeContentPlaceHolder>|Either the <CodeContentPlaceHolder>9\</CodeContentPlaceHolder> or the <CodeContentPlaceHolder>10\</CodeContentPlaceHolder> attribute is required.\<br />\<br /> The file path of the assembly. This attribute accepts full paths or relative paths. Relative paths are relative to the directory of the project file or targets file where the <CodeContentPlaceHolder>11\</CodeContentPlaceHolder> element is declared. Using this attribute is equivalent to loading an assembly by using the \<xref:System.Reflection.Assembly.LoadFrom*> method in .NET.\<br />\<br /> You cannot use this attribute if the <CodeContentPlaceHolder>12\</CodeContentPlaceHolder> attribute is used.|  
|<CodeContentPlaceHolder>13\</CodeContentPlaceHolder>|Optional attribute.\<br />\<br /> Specifies the class in the assembly that is responsible for generating instances of the specified <CodeContentPlaceHolder>14\</CodeContentPlaceHolder> name.  The user may also specify a <CodeContentPlaceHolder>15\</CodeContentPlaceHolder> as a child element that the task factory receives and uses to generate the task. The contents of the <CodeContentPlaceHolder>16\</CodeContentPlaceHolder> are specific to the task factory.|  
|<CodeContentPlaceHolder>17\</CodeContentPlaceHolder>|Required attribute.\<br />\<br /> The name of the task to reference from an assembly. If ambiguities are possible, this attribute should always specify full namespaces. If there are ambiguities, MSBuild chooses an arbitrary match, which could produce unexpected results.|  
|<CodeContentPlaceHolder>18\</CodeContentPlaceHolder>|Optional attribute.\<br />\<br /> The condition to evaluate. For more information, see [MSBuild Conditions](../vs140/msbuild-conditions.md).|  
  
### Child Elements  
  
|Element|Description|  
|-------------|-----------------|  
|[ParameterGroup](../vs140/parametergroup-element.md)|The set of parameters that appear on the task that is generated by the specified <CodeContentPlaceHolder>19\</CodeContentPlaceHolder>.|  
|[Task](../vs140/task-element--msbuild-.md)|The data that is passed to the <CodeContentPlaceHolder>20\</CodeContentPlaceHolder> to generate an instance of the task.|  
  
### Parent Elements  
  
|Element|Description|  
|-------------|-----------------|  
|[Project](../vs140/project-element--msbuild-.md)|Required root element of an [!INCLUDE[vstecmsbuild](../vs140/includes/vstecmsbuild_md.md)] project file.|  
  
## Remarks  
 Environment variables, command-line properties, and project-level properties can be referenced anywhere in the <CodeContentPlaceHolder>21\</CodeContentPlaceHolder> element if it appears in the project file either explicitly or through an imported project file. For more information, see [MSBuild Tasks](../vs140/msbuild-tasks.md).  
  
> [!NOTE]
>  Project-level properties have no meaning if the <CodeContentPlaceHolder>22\</CodeContentPlaceHolder> element is coming from one of the .tasks files that are globally registered with the MSBuild engine. Project-level properties are not global to MSBuild.  
  
 In MSBuild 4.0, using tasks can be loaded from .overridetask files.
